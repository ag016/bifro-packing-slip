# Contributing

Thank you for helping improve Bifro Packing Slips.

## Before opening a change

- Use an issue to describe significant changes before investing in a large implementation.
- Keep pull requests focused and explain the user problem, the approach, and how the change was verified.
- Add or update tests when behaviour changes.
- Never commit credentials, Apps Script IDs, customer records, personal information, or exports from a production Google Sheet.
- Report vulnerabilities privately by following [SECURITY.md](SECURITY.md).

By submitting a contribution, you confirm that you have the right to provide it and agree that it may be distributed under this project's [MIT License](LICENSE).

The maintainers may decline changes that add avoidable data collection, weaken customer ownership of the installed system, or create an ongoing dependency on Bifro infrastructure.
